Determine user tasks: CRA web service manual

To achieve your goals, you have to see the problem you want to solve from a user’s perspective.

Steps

  1. Define user tasks by using data and research to find out what users want to do with the information on your topic
  2. Then, use the GOV.UK user needs template to write user tasks:
    • As a… [who is the user?]
    • I need to… [what does the user want to do?]
    • So that… [why does the user want to do this?]
    Example: Good example
    • As a student who lost a job.
    • I need to find out what financial support is available to me.
    • So that I can continue paying for my studies and living expenses.

    Why this is a good example: The user is very specific and with unique life circumstances. The need of the user is related to basic needs. They need to have money to pay for their daily life and studies.

    Example: Bad example
    • As a person who lost a job.
    • I need to learn about the different types of emergency benefits offered by the government.
    • So that I can correctly submit the application.

    Why this is a bad example: The user is generic. People do not want to learn about the many different services the government offers. People may not know that financial support in a time of crisis is called an emergency benefit. They want to know what specific service will help them. The objective of the user is to get money so that they can pay for daily expenses, the format and process of how they get it is not important to them.
