Content approach

Purpose of panels

Use to create a visual grouping of content. Panels have colour themed options, as well as offer a distinct header and footer.

Correct use

General
  • Use panels to spotlight general information/actions within a self-contained box.
  • Use .panel-heading, .panel-title and .panel-body to spotlight general information/actions within a self-contained box
  • Ensure a heading <h*> is used within the .panel-heading
  • Use .panel for both full-width content, as well as content that is padded in from the border with .panel-body
Tables
  • Include a <caption> when the <table> is not directly under an <h*> tag
  • Use .wb-inv when it is necessary to make the caption invisible, this ensures accessibility
Colour
  • Use .panel-info to spotlight specialized information/actions
  • Use .panel-primary to spotlight general information/actions
  • Use .panel-success to spotlight correct information/actions
  • Use .panel-warning to spotlight warning information/actions
  • Use .panel-danger to spotlight dangerous information/actions
