Prepare an information return

This page includes information on the data fields and a high-level overview of the steps that platform operators should follow when filing an annual information return with the Canada Revenue Agency (CRA).

To file an information return, you need to represent your data in XML, in accordance with the Digital Platform Information Returns (DPIR) schema. For detailed information, go to the DPIR Schema User Guide and DPIR schema.

Each XML document consists of the following:

  • a header, described in the Message Specification data element in the schema
  • data about the platform operator
  • data about sellers, including their activities, earnings derived from those activities and related taxes

Please note that in the DPIR schema user guide, the terms XML document, DPIR message and message are used interchangeably and all mean the same thing.

Data fields for your information return

Data fields for the information return are organized into five groups:

  • platform operator data fields
  • entity seller data fields
  • individual seller data fields
  • data fields related to earnings from rental of short-term accommodation
  • data fields related to earnings from sales of qualifying tangible personal property

Listed below are the data fields for each group. For each data field, in brackets, the name of its associated XML data element in the schema is provided. This name links the data field directly to its XML data element.

For a complete description of data elements, see the DPIR schema user guide. For examples showing how to use them, see the examples link.

If you are missing information for any required data fields, consult the DPIR schema user guide. If the schema does not provide any guidance, please contact us at xxx and describe your situation.

Data fields about platform operators

Include the platform operator data fields in the Platform Operator (PlatformOperator) data element. These data fields are:

  • business number (TIN), called BN9 in the DPIR schema user guide
  • GST/HST registration number (VAT), called BN15 in the DPIR schema user guide
  • platform legal name (Name)
  • primary address (Address), which should be provided in structured format whenever possible
  • platform business name (PlatformBusinessName), if available
  • URL address (URL attribute), if available
  • tax residency country code (ResCountryCode), if available

Data fields about entity sellers

An entity seller is any seller, except for a sole proprietorship, that operates as a business. Sole proprietorships are categorized as individual sellers.

Include the data fields for identifying entity sellers in the Entity Seller ID (EntSellerID) data element. These data fields are:

  • business number (TIN), called BN9 in the DPIR schema user guide
  • GST/HST registration number (or VAT), called BN15 in the DPIR schema user guide
  • legal name (Name)
  • primary address (Address), which should be provided in structured format whenever possible
  • URL address (URL attribute), if available
  • tax residency country code (ResCountryCode), if available

For more information about these fields, including what to do if addresses and legal names are not available, see the DPIR schema user guide.

Data fields about individual sellers

Individual sellers operating as sole proprietorships may have a business number (BN9), a GST/HST registration number (BN15), or both.

Include the data fields for identifying individual sellers in the Individual Seller ID (IndSellerID) data element. These data fields are:

  • full name (Name); only first name and last name are mandatory
  • primary address (Address), which should be provided in structured format whenever possible
  • business number (TIN); if an individual does not have a business number (BN9), then use ‘NOTIN' and set the unknown attribute to "true"
  • GST/HST registration number (VAT), if available
  • tax residency country code (ResCountryCode) is a derived field based on the address

For more details about the above fields, including what do if full names or addresses are not available, please refer to the DPIR schema user guide.

Data fields about rental earnings from short-term accommodation

Include the data fields related to rental earnings from short-term accommodation in the Property Listing (PropertyListing) data element. These data fields are:

  • total annual earnings (Consideration) and related taxes (Taxes)
  • province or territory (Province attribute) where the rental property is located

For an example, please refer to the examples link.

Data fields about sales earnings for qualifying tangible personal properties

Include the data fields related to sales earnings for qualifying tangible personal properties with the Sale of Goods (SaleOfGoods) data element. The data fields are:

  • total annual earnings (Consideration) and related taxes (Taxes)
  • province or territory (Province attribute) in which the qualifying tangible personal properties were sold

For a specific example, please refer to the examples link.

Filing information returns

The following is general information on how to file your information return. The specific details for each step will likely be different for each platform.

Please note the CRA will update its website to inform platform operators when they can start filing their information return using the Internet File Transfer service.

Step 1: Identify the data fields to include in your return

Identify the data fields to include in your return, based on the type of service that your platform provides (either rental of short-term accommodation or sales of qualifying tangible personal property). Then, identify your data sources for those fields.

Step 2: Generate the Message Reference ID

Every XML document must have a unique identifier, which is called the Message Reference ID (MessageRefId).

A Message Reference ID consists of seven components, all of which are known at the time of filing except for the "message root." To create a message root, generate a six-digit number, either randomly or sequentially. This determines the unique Message Reference ID for the XML document and generates the identifier prefixes for the Platform Operator, Entity Seller and Individual Seller data elements.

For a detailed description of the Message Reference ID format, see the DPIR schema user guide.

Step 3: Export data into XML format

Export your data from the existing systems. Using a tool of your choice, transform the exported data into an XML document.

Report seller earnings and related taxes as follows:

  • for accommodation platform operators, report each seller's total short-term sales and related taxes (all properties rented by the seller) by province or territory
  • for distribution platform operators in respect of supplies of qualifying tangible personal properties, report each seller's total sales and related taxes by the province or territory in which the properties were sold

For examples, see the examples link.

Step 4: Perform final validation checks on the XML document

Final validation checks include making sure that:

  • the XML document complies with the DPIR schema
  • the Message Reference ID matches the Message Reference ID generated in Step 1
  • the format of Document Reference ID (DocRefId) of the Platform Operator complies with the format described in the DPIR schema user guide
  • the Document Reference IDs of Entity Sellers and Individual Sellers are unique and comply with the format described in the DPIR schema user guide

Step 5: Partition the XML document into several documents, if required

If the initial XML document produced in Step 4 is larger than 150MB, you must partition it into smaller XML documents that are no more than 150MB each. The partition must meet these requirements:

  • the set of Reportable Sellers (ReportableSellers) in the DPIR Body (DPIRBody) of the initial document must be partitioned into sets of Reportable Sellers in the DPIR Body of the smaller documents, so that each smaller document complies with the DPIR schema
  • each smaller document must have a unique Message Reference ID and Document Reference IDs of all corresponding Platform Operator and Reportable Seller data elements must be updated accordingly. For more information, go to the DPIR schema user guide.

Step 6: Upload the XML document

Go to File your return and upload the selected XML document. If you get any error messages, please make the necessary corrections.

Step 7: Record the reference number

After you upload the document successfully, a reference number will appear on the next page. Document and keep this reference number in your records. The CRA may ask for it if there are any subsequent problems with the transmission.

Step 8: Load the remaining XML documents (if any)

If you have more than one XML document, then repeat the previous three steps for the remaining documents. A separate reference number will be issued for each XML document.

Step 9: Keep a copy of the return you filed

After uploading each of your XML documents, keep a copy of each of them for your records.

If you get an error message when filing your information return

If you get an error message when filing your information return, please consult the following:

  • DPIR schema
  • DPIR schema user guide
  • Example data and resulting return
  • Modifying previously filed information returns

If the error messages persist after you adjust and re-file your return, contact us at XXX. Include your contact information and the error message you got.
