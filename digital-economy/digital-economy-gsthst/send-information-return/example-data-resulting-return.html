Example data and resulting return

Examples of Digital Platform Information

This page illustrate two examples in which Platform Operators use the DPIR schema to file information returns. The first example relates to sales of qualifying tangible personal properties, and the second example relates to rental of short-term accommodation.

Message Specification and Platform Operators data elements are common to both examples and are shown first.

Message Specification data element

Example 1 shows a Message Specification data element, where the Platform Operator is located in the United States, it has a BN15 of “123456789RT0001”, and is filing an information return for the calendar year 2022. The information return was filed on April 17, 2023, at 9:30 am.

<dpir:MessageSpec>
 <dpir:SendingEntityIN>123456789RT0001</dpir:SendingEntityIN>
 <dpir:TransmittingCountry>US</dpir:TransmittingCountry>
 <dpir:MessageType>DPIR</dpir:MessageType>
 <dpir:Contact>John Valore: 1800-1010-0111 ext. 3458, 8am-4am EST.
 For more info: john.valore@digitalcart2050ee.com</dpir:Contact>
 <dpir:MessageRefId> US2022CADPIR123456789M10000101</dpir:MessageRefId>
 <dpir:MessageTypeIndic>DPI401</dpir:MessageTypeIndic>
 <dpir:ReportingPeriod>2022-12-31</dpir:ReportingPeriod>
 <dpir:Timestamp>2023-04-17T09:30:47Z</dpir:Timestamp>
</dpir:MessageSpec>

Example 1 – Message Spec data element

The above message has a unique identifier "US2022CADPIR123456789M10000101" which follows the format specified in the DPIR schema user guide. The message identifier consists of the following components:

  • The Platform Operator is located in the United States and is filing an information return for the calendar year 2022
  • The information return is filed and transmitted to Canada, as part of the "DPIR" program
  • The Platform Operator's BN9 is "123456789"
  • The message root is "10000101"

Platform Operator data element

Example 2 shows an a Platform Operator data element. 

<dpir:PlatformOperator URL="http://www.digitalcart2050ee.com/">
  <dpir:ResCountryCode>US</dpir:ResCountryCode>
  <dpir:TIN issuedBy="CA">123456789</dpir:TIN>
  <dpir:VAT>123456789RT0001</dpir:VAT>
  <dpir:Name>Cart 1</dpir:Name>
  <dpir:PlatformBusinessName>Digital Cart Universe</dpir:PlatformBusinessName>
  <dpir:Address legalAddressType="OECD303">
	<dpi:CountryCode>US</dpi:CountryCode>
	<dpi:AddressFix>
	  <dpi:Street>San Antonio Boulevard</dpi:Street>
	  <dpi:BuildingIdentifier>76200A</dpi:BuildingIdentifier>
	  <dpi:PostCode>92408</dpi:PostCode>
	  <dpi:City>San Bernardino</dpi:City>
	  <dpi:CountrySubentity>California</dpi:CountrySubentity>
	</dpi:AddressFix>
  </dpir:Address>
  <dpir:DocSpec>
    <stf:DocTypeIndic>OECD1</stf:DocTypeIndic>
	<stf:DocRefId> US2022CADPIR123456789M10000101000001</stf:DocRefId>
  </dpir:DocSpec>
</dpir:PlatformOperator>

Example 2 – Platform Operator data element

Platform Operators will likely have both a BN9 (which is used as TIN) and a BN15 (which is used for a VAT), as Figure 2 shows. In the above example, the Platform operates under the business name "Digital Cart Universe", but is registered under the legal name "Cart 1". Figure 2 shows the URL of the Platform Operator.

The Platform Operator's business address (legalAddressType ='OECD303') is:

"76200A San Antonio Boulevard, San Bernardino, 92408, California, US".

Therefore, its tax residency country code is US.

The above example shows that the Document Reference ID of the Platform Operator is "US2022CADPIR123456789M10000101000001". The last six-digit message root (‘10000101') number does not have to start from zero and can be any number as long as it is distinct from any other message root sent from the same Platform Operator for the same Reporting Period.

Sales of Qualifying Tangible Personal Properties

Consider tabular data set in Table 1 containing transactional records of sales of qualifying tangible personal properties. The example includes Seller's details (both Individual Sellers and Entity Sellers), the province where the properties were sold, as well as the sales earned and the GST/HST collected. Values that should be reported are in green. The example demonstrates the following:

  • Sellers can be either Individual Sellers or Entity Sellers.
  • Both Individual Sellers and Entity Sellers may have a BN9 and a BN15.
  • A Seller registered for GST/HST purposes will remit the GST/HST (VAT) on its own. Platform Operators will collect and remit the GST/HST to the CRA on transactions by Sellers that are not registered for GST/HST.
  • Platform Operators report taxes and sales of only not registered Sellers.
  • GST/HST is calculated and collected based on the province where the property is sold. Different provinces have different GST/HST rates.
  • A Province value of UNKNOWN may be used only in exceptional circumstances, and the Platform Operator should always have a valid Province data element.

Sales are aggregated at the Seller level, by Province.

Rental of Short-Term Accommodation

This section illustrates an example of how to represent data on rental of short-term accommodation (shown in Table 2) in XML using the DPIR schema.

As in the previous section, the example in Table 2 shows both seller information and transactions of sales. For ease of exposition, we include only Individual Sellers and do not show Identity data elements. In general, both Individual Sellers and Entity Sellers may be engaged in rental of short-term accommodation.

The example in Table 2 shows a breakdown of sales by property and rental duration. Property details, such as property address, are not shown in the example. Only values in the green background colour are to be reported in the information return.

The example illustrates the following:

  • During a Reporting Period, a property may be rented on both a short-term basis (continuous rental duration of less than one month) and long-term basis (continuous duration of more than one month). Sales and taxes are aggregated separately for short-term rentals and long-term rentals, and only short-term totals are to be reported.
  • Taxes charged on a property are based on the province where the property is located. For example, Individual 1 rents two properties: one property is located in Ontario (where the GST/HST rate is 13%) and the other property is located in Alberta (where the GST/HST rate is 5%).
  • Platform Operators do not remit GST/HST on property rentals by Sellers who are registered for GST/HST. In Table 2, Individual 2 rents a property in Quebec (which has a GST/HST rate of 5%), but the Platform Operator does not remit GST/HST on this rental because the individual is registered for GST/HST.
  • Taxes and sales of registered Sellers do not have to be included in the information return.

We now illustrate how the above data is represented in XML, only as it relates to the Relevant Activities data element. The XML representation of Sellers is the same as in the case of sales of qualifying tangible personal properties, discussed previously.

<dpir:RelevantActivities>
<dpir:ImmovableProperty>
 <!-- short term sales of a rental in ONT-->
 <dpir:PropertyListing DurationClass="DPIR101" Province="DPIR005">
   <dpir:Consideration>
     <dpir:AnnualConsideration currCode="CAD">3551</dpir:AnnualConsideration>
   </dpir:Consideration>
   <dpir:Taxes>
     <dpir:AnnualTaxes currCode="CAD">462</dpir:AnnualTaxes>
     </dpir:Taxes>
 </dpir:PropertyListing>
 <!-- short term sales of a rental in ALB-->
 <dpir:PropertyListing DurationClass="DPIR101" Province="DPIR011">
   <dpir:Consideration>
      <dpir:AnnualConsideration currCode="CAD">616</dpir:AnnualConsideration>
   </dpir:Consideration>
   <dpir:Taxes>
      <dpir:AnnualTaxes currCode="CAD">31</dpir:AnnualTaxes>
   </dpir:Taxes>
 </dpir:PropertyListing>
</dpir:ImmovableProperty>
</dpir:RelevantActivities>

Example 7 – Relevant Activities data element (short-term accommodation)

Example 7 shows that each cell in the green background in the Total Sales and Total Tax columns in Table 2 is represented by Consideration and Annual Taxes data elements, respectively, with the Province attribute specifying the province where the rental property is located. The Duration Class attribute indicates that the property was rented on short-term basis (DPIR101).

The second Seller in Table 2 is registered for the GST/HST. The Platform Operator does not remit the GST/HST to the CRA, and does not include their sales and taxes in the information return. The Relevant Activities data element for such Sellers is empty, as the following figure shows.
