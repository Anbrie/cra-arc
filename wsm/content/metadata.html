Note: We are currently working on updating and aligning CRA web content metadata with the CRA metadata standards.

For this initial phase, we have prepared these guidelines to provide assistance to web content authors who are responsible for adding metadata to their pages.

Temporarily, until further notice, content authors do not need to provide custom elements, Audience, or Date Expired metadata for their content.

For guidance on adding metadata to a web page using Adobe Experience Manager, see the AEM web publishing guidelines for content providers

What is metadata for Canada.ca web pages?

Canada.ca metadata is information that describes content published on the Internet. It is the Internet-age term for information that librarians traditionally have put into catalogues.

Canada.ca metadata information consists of a set of attributes, or elements, necessary to describe the resource in question. As such, it is not usually displayed with the content, much like a library index card is not "part of" the book, it simply describes the book. It is still there, in the background, helping find and manage the information, but is generally not visible when accessing the content.

If you require more general information on metadata, refer to CRA Metadata Standards and the Appendix A : Metadata Catalogue.

If you require information on intranet metadata, refer to Intranet metadata.

Why is metadata important?

Canada.ca metadata information supports CRA web site users by improving findability and providing accurate, relevant information more quickly and efficiently allowing users to make informed decisions and access services faster.

Metadata improves the retrieval of web content by improving the accuracy of searchable content, as well as the performance of search engines by displaying more descriptive information on search result pages.

Canada.ca metadata assists CRA content providers and publishers manage CRA web content more efficiently and effectively.

Who is responsible for Canada.ca web pages metadata?

Content authors of CRA web pages are responsible for identifying metadata that is relevant and specific to their pages. As subject-matter experts, their knowledge and expertise are needed to ensure that metadata accurately reflects the content of the web page.

To which web resources does metadata apply?

As per the Treasury Board Secretariat (TBS) Standard on Metadata (external link), metadata should be applied to the following web resources:

  • Welcome pages and home pages (defined as the major entry point to an institution or those distinct organizational units where there is likely to be a public perception that they stand alone)
  • Topics or services in high demand by the community that the institution serves (this can be based on usage statistics but may also include topical or publicized resources with potential public interest)
  • Information required by the public to understand their entitlements to government assistance or obligations
  • Pages that provide an actual online service to the public (such as payment forms or application forms)
  • Pages required to meet a prescribed legal or service obligation of the institution
  • Entry points to specific online services and indexes (for example, an entry point to a legal database)
  • Major formal publications (for example, annual reports, corporate strategic plans, or public policy and accountability documents)
  • Media releases
  • Major entry points or indexes and menus to a range of closely related topics, programs, or policies
  • Information about agency powers affecting the public, and manuals and other documents used in decision making affecting the public
  • Substantial descriptive or marketing information about institutions, their services, activities, and collections

Note: Metadata can also be applied to other types of web resources, such as .pdf, .docx, and .pptx.

What are the metadata properties for Canada.ca web pages?

The Canada.ca metadata properties are defined by the CRA Metadata Standards and CRA Metadata Catalogue that came into effect on September 4, 2018. The standard requires that all information resources within CRA follow CRA Foundational Metadata Element Set and domain-specific metadata – in our case that is the CRA Web Content Metadata Element Set.

These are the mandatory metadata elements that should be applied to all pages on the Canada.ca website:

  • Title
  • Description
  • Subject
  • Keywords
  • Content type
  • License
  • Language
  • Date issued and Date modified
  • Creator

Optional elements available for use in AEM:

  • Primary Topic, Additional Topics
  • Audience
  • Date Expired
  • Geographic region name

Title

The name by which the web page is identified.

Points of compliance:

  • Create a unique title for a page
  • Titles should not exceed 200 characters, including spaces
  • Upper and lower case are required, because the title will display to users
  • Use proper capitalization, and include the following:
    • tax year,
    • acronym(s) and initialism(s),
    • registered document number, if applicable
  • Do not use curly apostrophe and quotation marks.
  • Do not use the vowel oe (use oe – 2 letters not ligature or use the character entity "œ" or "œ").

Title examples:

  • Paying your income tax by instalments
  • Agenda – Information Management Steering Committee – February 12, 2018
  • Savings and pension plans

Description

The summary of the page highlights key content to tell the user what information they can expect to find on the web page.

Note: Google, Bing, and other search engines may display the description on the search engine results page. Due to system configuration, the description is currently not shown in Canada.ca search results page, however it carries some weight in ranking.

Points of compliance:

  • Upper and lower case required; it is visible to users
  • Character limit is 400; however, best practice is to limit the description to 150 characters. Google truncates the description if displayed in the search engine results page, when longer than 150.
  • End with a period
  • Omit unnecessary words
  • Be as concise and as coherent as possible
  • Identify the most important information users will find in the document

Description examples:

  • Find information on when, why, and how to pay tax by instalments.
  • Information on what is metadata , when to use it, and how to write it so it effectively describes and helps users find intranet content.
  • Information about tax-free savings accounts, registered savings plans, pooled pension plans, and plan administrators.

Subject

The topic, content, or category of the web page.

Points of compliance:

  • Selected from the GC Core Subject Thesaurus, values are organized by subject, from general to specific
  • Select the most appropriate term (minimum of one) that is relevant to the content of the page. Example: Economics and Industry > Taxes > most appropriate term (e.g. Income tax) > most appropriate term if applicable (e.g. Personal income tax)

Keywords

Commonly used words or phrases that describe content on the web page.

Note: Keywords are not a ranking factor for most external search engines such as Google; however, they do carry weight in terms of ranking in the Canada.ca search engine. Use relevant words that best describe the content and topic. Avoid using vague terms that will bring up 1000's of search results.

Points of compliance:

  • Use keywords that users are likely to use when searching for the topic
  • Do not use vague terms, be specific, and use keywords that directly relate to your content:
    • Extract them directly from the document, if possible
  • Most relevant keywords should be at the beginning
  • Use acronyms that are part of CRA programs or applications (such as TFSA, GST/HST, RRSP, etc.), if relevant to the content
  • Separate keywords with commas ( , )
  • Do not use double quotation marks ( " )
  • Do not send keywords to Translation Services
    • Pick English words from the English document
    • Pick French words from the French document
  • Do not exceed 400 characters, including spaces

Keywords examples:

  • Instalments, how to pay by instalments, instalments schedule, periodic payments, paying taxes, instalments dates,
  • web content, metadata properties, how to write metadata, creating keywords, writing descriptions, creating keywords, search terms, metadata tags,
  • TFSA, RSP, RRSP, tax-free savings accounts, registered savings plans, pooled pension plans, registered education savings plans, RESP, RDSP, tax free savings,

Content type

A web-specific subset of the GC Resource Type, describing the nature or genre of the content.

Points of compliance:

  • Choose the most appropriate value relevant to the content.

License

An applicable license for the page.

Points of compliance:

  • Use "Open Government License"

Language

The language of the content of a web page.

Date issued and Date modified

Use ISO 8601 (external link) standard: (YYYY-MM-DD).

Points of compliance:

  • Date issued is the date that the content of the resource was first posted on the web.
  • Date modified is the most recent date on which the document was substantially changed and reposted.
  • The content attribute of the Date Issued and Date Modified cannot be left blank.

Creator

Identifies the organization/entity/position or person responsible for creating the content of the resource.

Points of compliance:

Added automatically based on user access to AEM.

Optional elements available for use in AEM:

Primary Topic, Additional Topics

A controlled list of applicable topics for the page.

Audience

Always choose "general public".

Date Expired

The date when the page's content will cease to apply.

  • Leave blank if unknown.
  • All campaign pages should have a Date Expired set.

Geographic region name

Always leave the default as "Canada".

What is intranet metadata?

Intranet metadata is information about content you publish on the intranet. Use it for maintenance, management and discovery of content on the intranet.

Some metadata properties are visible to users to help them find and distinguish different types of information. Other metadata properties are visible only to those who take care of the intranet. They help remove outdated content and provide important information about who is responsible for maintenance and updates to this content.

To view metadata for any intranet page do the following:

  1. Select the View button from the top toolbar in your internet browser
  2. Select Source (or Page Source)
    • A window opens and displays the source code  
    • The metadata is near the top of the page within the opening and closing <head> tags
    • It starts with <meta …

If you require more general information on metadata, refer to Information Management: Metadata.

If you require information on internet metadata, refer to Writing Metadata for Canada.ca web pages.

Why is intranet metadata important?

Intranet metadata is important for two reasons:

  1. Title, description and keywords metadata properties  help the search engines rank and prioritize information so that users can easily find what they are looking for
  2. Creator, language, date, and date range metadata properties help the intranet program manage intranet content  for the Agency

When do you need intranet metadata?

You always need some metadata for intranet content. The amount of metadata required depends on whether the page you are publishing is new or updated, indexed or not indexed.

Note: The search engine should not find (rank) all content you publish. In other words, do not index all content. Refer to intranet indexing guidelines for pages types that require indexing.

Apply the following metadata for new content that is not indexed:

  • Title
  • Creator
  • Publisher
  • Language
  • Date
  • Date range
  • Robots

Apply the following metadata for new content that is indexed:

  • Title
  • Creator
  • Publisher
  • Language
  • Date
  • Date range
  • Robots
  • Description
  • Keywords
  • Owner

Apply the following metadata for updated content (both indexed or not):

  • Date
  • Date range
  • Creator (if it requires modification)
  • Robots (if it requires modification)

Who is responsible for intranet metadata?

The content subject matter expert (usually the content author) is the best person to write the metadata.

General rules around intranet metadata

When you create metadata:

  • Have a strong understanding of your content and its intended audiences
  • Write metadata in lower case only – except for proper names or acronyms

For further help with intranet metadata, refer to Writing Metadata for the CRA Intranet - TD3503-000 course

Intranet metadata properties

Title

Purpose: It lets users see the title in search returns.

Points of compliance:

  • Create a unique title for a page
  • Use the exact same text for the three title tags
    • This ensures search result links, bookmarks, web page titles and main headings are consistent
    • The three tags are:
      1. dcterms.title metadata
      2. <title> HTML tag
      3. <h1> HTML tag
  • Use proper capitalization, and include the following:
    • tax year,
    • acronym(s) and initialism(s),
    • registered document number,  if applicable
  • Do not exceed 200 characters, including spaces

Title examples:

  • <meta name="dcterms.title" content="Intranet metadata" />
  • <title>Intranet metadata</title>
  • <h1>Intranet metadata</h1>

Description

Purpose: In search results, it displays a summary of the page content.

Points of compliance:

  • If the page has task, organizational, or content description teaser text, use it as description metadata
  • If the page does not have task, organizational, or content description teaser text, create one to  use as description metadata by following teaser text instructions

Description examples:

  • <meta name="dcterms.description" content="Information on what is metadata, when to use it and how to write it so it effectively describes and helps users find intranet content.." />

Keywords

Purpose: Keywords enable users to find what they are looking for when they use a search box. As a content author you need to write metadata keywords the user, not you, is likely to use in a content search.

Points of compliance:

  • Use keyword terms that directly relate to your content
    • Extract them directly from the document, if possible
  • Use upper or lower case, and accented characters where applicable
  • Do not use double quotation marks ( " )
  • Separate keyword terms by semicolons ( ; ) not commas ( , )
    • You can alphabetize your keywords to make management easier
  • Do not send keywords to Translation Services
    • Pick English words from the English document
    • Pick French words from the French document
  • Do not include variations of a word (singular / plural, alternate spellings)
    • You do not need to re-enter words in phrases as single terms
  • Do not exceed 400 characters, including spaces

Keywords example: <meta name="keywords" content="intranet authors; web content; Dublin Core; metadata properties; how to write metadata; creating keywords; search terms; metadata tags; writing metadata" />

Creator

Purpose: To identify the organization responsible for creating the content of the resource.

Points of compliance:

  • Start with the largest organizational unit and end the list with the smallest organization

Creator example: <meta name="dcterms.creator" content="Government of Canada, Canada Revenue Agency, Public Affairs Branch, Publishing Directorate, Agency Intranet Program" />

Owner

Purpose: The owner metadata property is like dcterms.creator. It identifies who owns the content. It is used mainly for search engine results.

Points of compliance:

  • Include only the branch acronym or region name, plus name of the directorate, program, committee, tax centre or tax services office

Owner example: <meta name="owner" content="PAB, Publishing Directorate" />

Publisher

Purpose: Identifies the organization that publishes or posts the material to the intranet.

Points of compliance:

  • Start with the largest organizational unit and end the list with the smallest organization

Publisher example: <meta name="dcterms.publisher" content="Government of Canada, Canada Revenue Agency, Public Affairs Branch, Digital Design and Production Directorate, Production Services, Web Development" />

Language

Purpose: Identifies the language of the resource and conforms to a three-letter standard abbreviation.

Points of compliance:

  • Language abbreviations must come from the proper name of a foreign language instead of the English name ( ISO 639.2T language standard (External link))

Language examples:

  • French: <meta name="dcterms.language" title="ISO639-2" content="fra" />
  • English: <meta name="dcterms.language" title="ISO639-2" content="eng" />

Date

Purpose: Two date tags are included in the standard CRA intranet template: dcterms.issued and dcterms.modified.

  • The dcterms.issued metadata represents the date a resource is first posted to the web
  • The dcterms.modified metadata indicates the date a resource is significantly changed
  • The content attribute of the dcterms.issued and dcterms.modified cannot be left blank
    • Until any substantial revisions to the page take place, use the dcterms.issued value for dcterms.modified

Points of compliance:

  • Use the YYYY-MM-DD standard
  • Update the dcterms.modified tag for major content changes
  • Do not update dcterms.modified for typo corrections

Date example:

  • Issued: <meta name="dcterms.issued" title="W3CDTF" content="2010-02-14" />
  • Modified: <meta name="dcterms.modified" title="W3CDTF" content="2010-02-15" />

Date range

Purpose: dcterms.valid assigns a valid date range for content. Providing a date range will prompt more consistent review and management of content to prevent accumulation of ROT content.

Points of compliance:

  • Use the YYYY-MM-DD standard
  • Keep the range can be as short as needed, but the maximum range is 5 years
  • Apply (always) to any new or updated content

Date range example: <meta name="dcterms.valid" title="W3CDTF" content="2015-08-17/2020-07-31" />

Robots

Purpose: The robots metadata property shows or hides a page and its linked pages from search engines. Hiding some intranet pages provides better results by reducing search clutter. By default, the robots tag is set to hide a webpage from the search index using noindex,follow ("noindex" means that the search engine will not retrieve or display content).

Note: The nofollow value for the Robots metadata element is not used on InfoZone intranet pages.

Points of compliance:

  • Refer to the intranet indexing guidelines for page categories that require indexing in the search engine (index,follow)
  • Use the "index, follow" value if you want a search engine to find and list your page in search results

Robots examples

  • Exclude page from the search index: <meta name="robots" content="noindex,follow" />
  • Include page in the search index: <meta name="robots" content="index,follow" />

System metadata

These mandatory fields are input by AIP webmasters and allow the site to function as intended. Content authors do not need to provide system metadata.

Parent URL

Purpose: The parent URL metadata identifies the parent of the current page in the site structure. This field affects the breadcrumb of the page.

Parent URL example: <meta name="parentUrl" content="/english/r1713497/rsc/cntnt-e.html" />

Language link

Purpose: The language link metadata identifies the corresponding page in the other official language, or directs the browser to a notice for unilingual pages.

Points of compliance: refer to language link in the WET 4 templates

Language link examples:

  • Bilingual page example:
  • <meta name="languagelink" content="/francais/r1713497/org/sa/sa06/sb/sb01/tlr/cnt5/index-f.html" />
  • <meta name="languagelinktext" content="Français" />
  • Unilingual page example:
  • <meta name="languagelink" content="/gbl/avertissement.html" />
  • <meta name="languagelinktext" content="Avertissement" />

Feedback

Purpose: The feedback metadata populates a link that people may use to provide feedback about the page. This link may be generic to InfoZone or area-specific.

Points of compliance: refer to feedback buttons

Feedback examples:

  • InfoZone: <meta name="feedback" content="/english/frames/feedback-e.html" />
  • Branch-specific: <meta name="feedback" content="/english/r2732472/fdbck-e.html" />

Missing metadata message

Hello,

You recently submitted content without any metadata and indicated that you want it to be found by the search engine. Since metadata supports search results and end users who might look for your content, the lack of metadata impacts users' ability to find your webpage and where it appears in search results. You can use the online course TD3503-000 Writing Metadata for the CRA Intranet for instructions about how to write metadata that will make it easier to find your content.

For any questions about metadata, please email InfoZone.

Thank you.
